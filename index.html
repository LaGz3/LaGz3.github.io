JSP学习

Web 应用程序开发基础知识

  1. Web结构
    Web主要由两个部分组成，提供信息的 Web 服务的服务器端网站及向网站提出信息内容浏览要求的客户端浏览器。
    要打开和浏览网络上的网页文件，必须通过浏览器程序，而使用浏览器打开网页的这一端，称为客户端
    与客户端相比，提供浏览网页服务的一方称作“服务器端”，而用来放置这些网页信息的计算机，则称为服务器，服务器的功能并不只是单纯地存放网页信息，任何可提供网络服务的计算机都是服务器，例如提供网页信息的称为网页服务器，而提供文件上传与下载功能的则称为文件服务器
  2. 网页的分类
    静态网页：是指一旦网页制作成功就不能随意更改或需要专业的技术人员进行修改
    动态网页：是指能够根据用户的请求而动态生成显示的结果。
  3. 客户端动态网页技术
    客户端的浏览器在接收到服务器端传送而来的资源时，还需要负责对其中的脚本进行解释执行，并产生动态效果（可用JS或VBs脚本语言）
  4. 服务器端网页的运行过程
    (1) 用户通过浏览器指定 URL，向网页服务器请求特定的网页内容。
    (2) 服务器加载指定的 JSP 网页，通过解释器解读其中的 JSP 程序代码，创建HTML 网页，然后返回至客户端的网页浏览器进行解释。
    (3) 浏览器取得 JSP 文件，解释其中的 HTML 及 Script 网页程序代码。
    (4) 显示网页。
    在上述过程中，用户可以发现服务器端的网页语言与 HTML 静态网页最大的差异在于第 2 个步骤，这个部分通过执行 JSP 程序代码创建 HTML 网页，因此在不同条件的情况下，网站得以利用同一份 JSP 网页文件，输出完全不同的网页内容，甚至动态产生 Script 程序代码。
  5. 什么是JSP
    JSP 技术是在传统的网页 HTML 文件(.htm,.html)中插入 Java 程序段 (Scriptlet) 和 JSP 标记 (tag)，从而形成 JSP 文件(*.jsp)。
  6. Servlet概念
    Servlet是运行在服务器端上的Java程序，是一个中间层，主要用来处理客户端请求，并对客户端请求进行响应，还可以连接应用服务。
    Servlet 是在服务器端实现动态网页的一种技术
  7. JavaBean
    JavaBean 用来包装网页程序逻辑及其相关功能，并且通过网页作引用，而网页本身则负责数据收集和内容展示。
